C'est l'histoire d'un projet ...

Analyse du besoin

  • Contextes métier isolés : Résultats d'analyse, Annuaire, Données de référence
  • Imports : gros volume de données, source FTP, besoin de performance, utilisation bornée dans le temps
  • Reporting : stockage des résultats d'imports (erreurs, mises à jour, créations ...), génération de documents (pdf, excel ...), notifications, déclenchement suite import
  • Exports : vers d'autres SI, vers les différents intervenants

Analyse du besoin

  • Consultation : au fil de la journée, readonly pour certains contextes du métier
  • Volumétrie : importante en import, export et stockage des résultats d'analyse. Prévision en hausse.
  • Évolutivité : échéances à court et moyen termes

Déductions

  • Besoins de performance variables en fonction des contextes métiers
  • Sécurisation des données entrantes
  • Échanges entre services : import => reporting => notification

Microservices ?

Pros

  • Scalabilité indépendante de chaque service, au besoin
  • Bases de code réduites, rapidité de montée en compétence sur un domaine
  • Facilité de maintient, évolution, débug ...

Cons

  • nécessite des équipes de développement importantes / scindées
  • base de code scindée => MAJ de code commun coûteuses ...
  • requêtes BDD deviennent requêtes HTTP (44 Mo d'erreurs ????)
